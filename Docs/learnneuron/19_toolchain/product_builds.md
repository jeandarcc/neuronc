# Product Builds

Neuron includes a built-in production build system designed to convert your (`.nr`) sources into distributable desktop applications with a single command. 

---

## The Command

```bash
neuron build-product --platform Windows
```

This command orchestrates the entire release pipeline:
1. **Compiles NCON Container**: Compiles your project into an optimized `.ncon` bytecode container.
2. **Generates Executable**: Builds a minimal native runtime for the target platform (e.g., `MyApp.exe`).
3. **Embeds Resources**: Compiles platform-specific application icons and version information (e.g., via `windres` on Windows).
4. **Self-Extracting Binary**: Embeds the `.ncon` payload directly into the binary.
5. **Version Bumping**: Automatically increments the build version in your `.productsettings`.
6. **Manifest Generation**: Generates a `product-manifest.json` with metadata for release tracking.

The output is written to `build/product/<version>/` by default.

---

## Configuration (`.productsettings`)

The build pipeline is customized using the `.productsettings` file located at the root of your project. This file is automatically generated when you create a new project using `neuron new`.

```ini
# ── Identity ──
product_name = "My Application"
product_version = "1.0.0"
product_build_version = 1
product_publisher = "Publisher Name"
product_description = "A Neuron application"
product_website = "https://example.com"

# ── Branding ──
icon_windows = "assets/icon.ico"
icon_linux = "assets/icon.png"
icon_macos = "assets/icon.icns"

# ── Output ──
output_name = "MyApp"
output_dir = "build/product"

# ... and more Installer/Updater configurations
```

**Auto-Increment**: Every successful product build automatically bumps the `product_build_version`. The physical output folder name corresponds to `product_version` to keep your builds organized by semantic versioning.
